# Blog-Hexo

## 文件

博客文章在 `source/pages` 目录下
静态资源在 `source/static` 目录下

## 贡献

1. `fork` 本仓库到自己的账号下
   
2. `clone` 自己的仓库到本地

3. 修改或添加 `source/pages` 目录中的文件
   图片或其他静态资源, 放在 `source/static` 目录下, 你可以新建一个与Mrakdown文件名相同的文件夹用于放静态资源

4. 无需手动编译HTML和部署, GitHub Action会自动编译, 自动更新Cloudflare CDN
